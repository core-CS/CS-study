## Project Deep Dive Questions

본인의 프로젝트를 30초 내로 설명해주세요 (미사여구 없이).

왜 이 문제를 선택했습니까? 어떤 대안들이 존재했습니까?

본인 시스템에서 가장 취약한 부분은 어디입니까?

갑자기 10,000명의 사용자가 접속한다면 무엇이 가장 먼저 망가질까요?

아키텍처 설계에서 어떤 트레이드오프(trade-offs)를 고려했습니까?

SQL 대신 MongoDB를 선택한 이유는 무엇입니까? 특정 시나리오를 들어 정당화해주세요.

DB가 느려지면 정확히 어떤 조치를 취하시겠습니까?

사용자가 로그인할 때 내부적으로 어떤 일이 발생합니까?

본인 프로젝트에서 기술적으로 가장 자랑스러운 부분은 무엇입니까?

구성 요소 하나를 제거한다면 어떤 기능이 실패합니까?

본인 시스템에서 확장이 불가능한 부분은 어디입니까?

본인 시스템의 가장 큰 병목 지점(bottleneck)은 무엇입니까?

시간이 주어진다면 무엇을 다시 설계하시겠습니까?

시스템의 지연 시간(latency)을 어떻게 더 줄이시겠습니까?

본인이 해결한 가장 복잡한 버그는 무엇입니까?
