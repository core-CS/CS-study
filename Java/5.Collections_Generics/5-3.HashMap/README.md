## HashMap의 내부 동작 원리와 해시 충돌(Collision) 발생 시 자바에서 이를 해결하는 방법(Separate Chaining 등)은 무엇인가요?

> 💡 해시맵은 해싱 알고리즘과 key-value 구조를 지닌 맵을 섞은 자료구조입니다.
> 먼저 hashCode()를 통해 인덱스를 결정하고, 해시 충돌 시 separate chaining 을 통해서 linked list 로 연결합니다.
> 만약 데이터 전체 크기의 75% 이상이 차게 된다면, 리사이징을 하고 재배치를 합니다.
> 

### HashMap 이란?

***

`Map` 인터페이스를 구현한 자료구조입니다.

그냥 Hash 와 Map의 구조를 모두 합한 자료구조라 생각하면 되는데,

> **Map** = Key, Value 쌍으로 이루어지며, 순서를 보장하지 않고, 키는 중복이 불가하나 value 는 중복이 가능하다.

> **Hash** = 해시 함수를 사용해 임의의 길이를 가진 데이터를 고정된 길이를 가진 데이터로 매핑한 값

**Key-Value** 쌍의 데이터를 저장하고, **Hashing** 알고리즘을 사용해서 데이터 위치를 결정합니다.

> 그래서 순서를 보장하지 않습니다.

| Key | Value | 
| -- | --- |
| 0 | 010-1234-1234 |
| 1 | 010-9091-9091 |
| 2 | 02-1938-2049 |
| ... | |

위처럼 저장이 될 때 인덱스에 Key를 넣어 관리를 할 수 있습니다.

Key는 정수가 아닌 모든 타입의 인스턴스가 들어올 수 있는데,

배열의 인덱스를 키로 활용 한다는 관점에서 정수가 아닌 값이 들어오는 게 이상하다고 생각할 수는 있는데,

> 키 값 -> 해시함수 -> 정수(이를 배열의 인덱스로 사용)


참고로

`null` 키와 `null` 값이 가능합니다.

### 단계별 내부 동작 원리

***

먼저 이를 알기 전에 HashMap 내에는 Node 타입 변수가 선언되어 있는데, 이에 배열이 저장된다.

```java
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;
}
```

멤버 필드로 **hash, key, value, nextNode**가 존재한다.

> 왜 key와 value 만 있는 것이 아닐까? 

이 둘 값만 저장하면 여러 가지 문제가 발생하는데, 이 중 하나가 해시 충돌 (Collision) 과 리사이징, 해시 재배치 등의 문제가 있다.

1️⃣ Hashing & Index 결정

Hash 코드를 호출해 key 를 기준으로 인덱스를 계산한다.

2️⃣ 리사이징

> 새로운 길이의 배열을 생성한 후 데이터를 이관시켜 결과적으로 **배열의 사이즈를 변경시키는 작업**

- 배열의 사이즈는 초기화 시에 정해지는데, 기본 사이즈는 **16사이즈의 노드 배열**이다.

- 그리고 참고로 리사이징 임계 값은 12이다. (0.75*16 이라네요.)

> 임계값을 초과한다면 원래 사이즈를 2배로 늘리는 리사이징 작업이 알아서 수행된다.


3️⃣ 재배치

> 배열의 사이즈가 변경될 때 기존 데이터들의 **인덱스를 재배치하는 작업**

- 리사이징을 하면, 기존 노드들의 인덱스가 재배치되어야 한다.

- 우리가 Index 를 구하는 공식인 hashCode `% Size` 이기에 Size 가 수정되어서 Index 또한 바뀐다.

- 우리가 재배치 때 해시 값을 가져와서 연산을 해야 하는데, 해시 값을 위 노드에서 저장했기에 나머지 연산만 하면 된다.

🤔 이게 무슨 소리니?



### HashMap 내부 동작에서 발생할 수 있는 문제점과 해결법

***

💥 해시 충돌 (Collision)

지금은 우리가 인덱스 기반으로 값을 넣는데, 이 index 가 충돌할 확률은 어떻게 될까?

-> 우리는 사이즈에 대한 나머지를 인덱스로 사용하기에

-> 배열 사이즈가 20 이라면, 인덱스 중복 확률은 1/20 이 된다.

```
[예시]
- hash size 가 13이고,
- `(key, value)` 가 (1, 99), (14, 89)를 넣으려 한다.

Key가 1과 14는 동일한 해시 값을 가지게 되어 인덱스가 충돌된다.

```


🔁 충돌 위회 전략 (Separate Chaining)

> 동일 해시값이 이미 존재할 경우 `LinkedList` 로 관리한다.
>
> 즉, 노드 안의 nextNode 를 통해 중복 해시 값에 대해 Value 로 관리한다.

그렇기 때문에 위에서 보았던 `nextNode` 가 존재하는 것이다.

> 🤔 그런데 만약 아예 동일한 값이 들어온다면?

```
[예시]
- hash size 가 13인데,
- `(key, value)` 가 (1, 99), (14,99)를 넣으려 한다.
```

다른 value 라면 linked list 로 연결하면 되지만,

그런데 인덱스가 동일하다면 실제 Key 도 확인해야 한다.

그렇기 때문에 Key 도 저장하는 것이다.
