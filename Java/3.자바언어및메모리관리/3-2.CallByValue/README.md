## 자바의 Call-By-Value 방식에 대해 설명하고, 기본 타입과 참조 타입의 전달 방식 차이를 서술해주세요.

> 자바는 항상 **Call-By-Value** 로 작동됩니다.
>
> 기본 타입의 경우 실제 값이 복사되어 전달되므로 메서드 내 수정이 호출자에게 영향을 주지 않습니다.
>
> 참조 타입 역시 객체의 주소 가 복사되어 전달됩니다.
>
> 이 때 복사된 주소값을 통해 객체의 내부 필드를 수정하면 원본 객체도 영향을 받지만, 매개변수 자체에 새로운 객체를 할당하는 것은 원본 변수가 가리키는 주소를 바꾸지는 못합니다.


자바는 오직 Call-By-Value 방식만 사용합니다.

### 1️⃣ Call-By-Value

메서드를 호출 시 인자로 전달되는 변수를 **그대로 복사**해서 넘겨주는 방식입니다.

호출된 메서드 안에서 복사본을 아무리 수정해도, 호출한 쪽의 **원본 변수는 변하지 않습니다.**

### 2️⃣ 타입별 전달 방식 차이

🅰️ 기본 타입

변수 **스택 메모리**에 실제 값이 들어 있습니다.

- 메서드 호출 시 실제 값이 복사됩니다.

- 메서드 내부에서 값을 변경해도 원본 변수에는 전혀 영향이 없습니다.

🅱️ 참조 타입

변수 **스택 메모리**에 객체의 주소가 들어 있습니다.

- 메서드 호출 시 주소값이 복사됩니다.

- 복사된 주소로 **객체 내부 필드를 수정**한다면 원본과 복사본이 같은 객체를 가리키고 있으므로 **원본도 수정**됩니다.

- 그러나 Call-By-Reference 가 아닙니다.!!

- 우리가 이 메서드 내부에서 매개변수에 `new`로 할당하면, 복사된 주소 변수만 새 객체를 가리키게 될 뿐, 원본 변수는 여전히 기존 객체를 가리킵니다.


```java
public class CallByValueTest {
    public static void main (String[] args) {
        int a = 10;
        User user = new User("홍길동");

        modify(a, user);

        // result = 10
        System.out.println("a: " + a);

        // result = "김철수"
        System.out.println("user: " + user.name);
    }

    public static void modify(int 원래값, User user) {
        원래값 = 20; // call-by-value

        user.name = "김철수"; // 복사된 주소로 객체 접근하기에 원본이 수정

        // 매개 변수 자체에 새 주소를 할당해도 영향이 없다.
        user = new User("신짱구");
    }
}

```

🚨 주의 사항

참조 타입을 넘길 때 메서드 안에서 객체 상태가 변할 수 있음을 항상 인지해야 합니다 !

방지하고 싶다면 `final` 키워드를 쓰거나 객체를 불변으로 설계해야 합니다.

또한 비즈니스 로직에서 원본 엔티티의 값이 오염되는 것을 막기 위해, 데이터를 복사한 DTO를 넘기는 것이 안전합니다.
