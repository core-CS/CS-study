## `final` 키워드를 변수, 메서드, 클래스에 사용할 때 각 어떤 효과가 있으며 사용시 어떤 이점이 있나요?

> `final` 키워드는 불변성과 설계 의도를 명확히 해줍니다.
>
> 변수에 사용 시 초기화 후 변경이 불가능해 상수 또는 불변 객체로 사용됩니다.
> 메서드에 사용 시 하위 클래스의 오버라이딩을 막아 로직의 변경을 막고, 클래스에 사용 시 상속 자체를 금지해 확장 및 변경을 막아줍니다.
>
> 이를 통해 멀티 쓰레드 환경의 안정성을 확보할 수 있고, 코드의 예측 가능성을 높여 유지보수에 도움이 됩니다.


final 키워드는 **변경 불가능**을 선언하는 도구입니다.

## 위치별 효과

### 1️⃣ 변수 (Variable)

변수에 `final` 이 붙으면 한 번 초기화된 값을 다시 바꿀 수 없습니다.

- 기본 타입: 변수에 담긴 값 자체를 다시 바꿀 수 없습니다.

- 참조 타입: 변수가 가리키는 객체의 주소값을 바꿀 수 없습니다. (단, 객체 내부의 필드 값은 변경될 수 있습니다 !)

컴파일 시점에 재할당 시도를 체크해 에러를 발생시킵니다.

### 2️⃣ 메서드 (Method)

자식 클래스에서 해당 메서드를 Override 를 할 수 없게 합니다.

- 부모 클래스에서 정의한 로직이 자식에서 변할 수 없고 항상 동일하게 실행되도록 보장합니다.

### 3️⃣ 클래스 (Class)

해당 클래스를 부모로 하는 자식 클래스를 만들 수 없습니다.

`String`이나 `Integer`이 final 로 선언되어 있습니다.

```java
// 1. final class : 이 클래스를 확장하거나 변경 불가능
public final class SecurityConfig {

    // 2. final val : 상수로 사용
    public static final int MAX_LOGIN_ATTEMPTS = 5;

    // 3. final method: 하위 로직에서 변경 불가능
    public final void validateToken(String token) {
        tokenUtil.validate(token);
    }
}

```

##  사용 시 이점

1. 불변성과 안정성

- `final` 변수를 활용하면 객체가 생성된 후 상태가 변하지 않는 **불변 객체**를 만들기가 쉬워집니다.

- 멀티쓰레드 환경에서 여러 쓰레드가 동시 접근해도 데이터가 수정될 걱정 없는 Thread-Safe한 코드를 만들어줍니다.

2. 가독성과 유지보수성

- 코드를 읽는 사람이 이 `final` 키워드 하나만으로 중간에 값이 바뀔일이 없다는 것을 예측 가능합니다.

3. 컴파일 최적화

- 컴파일러는 해당 값이 변하지 않는다는 것을 미리 알기에 더 효율적인 기계어 최적화가 가능합니다.
