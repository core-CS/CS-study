### **Q. 함수형 인터페이스(Funtional Interface)란 무엇이며, 람다식과의 관계는 어떻게 되나요?**

함수형 인터페이스는 추상 메서드가 단 하나 존재하는 인터페이스를 말합니다.

람다식은 인터페이스의 메서드를 구현하는 역할인데, 만약 인터페이스의 추상 메서드가 여러개라면 람다식은 어떤 메서드를 구현해야 하는지 알 수 없기 때문에 컴파일 에러가 발생합니다.

따라서 람다식은 반드시 함수형 인터페이스를 기반으로 사용됩니다.

</br>
</br>

**함수형 인터페이스**

- 추상 메서드가 하나만 존재하는 인터페이스
- 추상 메서드 : 구현이 없는 메서드 ⇒ 틀만 정의하고 실제 구현은 나중에 하기 위해 (인터페이스의 메서드는 기본적으로 추상 메서드)
- `@FunctionalInterface` 어노테이션 추가해서 구현
    - Q. 해당 어노테이션을 꼭 추가해야 되나요? 그냥 인터페이스에 메서드 하나 적으면 안되나요?
    - 어노테이션을 추가하면 컴파일러가 메서드가 하나인지 검증해줘서 안정성과 의도를 명확히 할 수 있음

```java
@FunctionalInterface
interface MyFunction {
    void run();
}
```

**→ 람다는 항상 함수형 인터페이스를 기반으로 사용된다**

```java
interface MyFunction {
    void run();
    void call();
}

public class Main {
    public static void main(String[] args) {
        MyFunction f = () -> System.out.println("Hello");
        // 🚨 컴파일 에러 : run을 구현할지, call을 구현할지 모호함 
    }
}
```

만약 인터페이스에 추상 메서드가 2개 이상이라면,

람다는 어떤 메서드를 구현해야 하는지 모호해지기 때문에 사용할 수 없다.

따라서 람다는 항상 하나의 추상 메서드만 가지는 함수형 인터페이스를 기반으로 사용된다.
