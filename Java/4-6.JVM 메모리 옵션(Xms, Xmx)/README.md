## JVM 메모리 옵션인 Xms와 Xmx의 의미와 설정 시 고려해야 할 사항은 무엇인가요?
### -Xms와 -Xmx
Jar 파일을 실행할 때 초기 할당 메모리와 최대 할당 메모리를 지정하는 옵션

> -Xms2048m -Xmx4096m

이와 같이 설정해주었다면 초기 힙 메모리 사이즈가 2048MB이고 최대는 4096MB로 설정이 됨!

### heap 메모리를 잘 설정해야하는 이유
1.  gc
    최대 Heap메모리 사이즈가 작게 설정이 되어 있으면, GC가 그만큼 자주 발생할 것이고 이는 성능적으로 문제가 될 수 있어서 가능하면 넉넉하게 Heap메모리를 설정
2. 초기 heap 메모리와 최대 heap 메모리
   Heap메모리가 부족하면, 최대 Heap메모리 사이즈가 될때까지 추가적으로 메모리 사이즈를 할당

   예를 들어서 초기 Heap사이즈를 1G 최대 Heap사이즈를 4G로 설정했다고 해보자. 서비스가 구동되면서, 1G가 넘는 데이터가 Heap메모리에 할당이 된다면, 필요한만큼 Heap메모리 크기를 늘리게 됨!

    과정 자체는 정상적인 것이지만, 문제는 이 과정이 일어나면 GC가 동작하는 것이다.

    즉, 추가 Heap을 요청하는 행위가 빈번하게 발생할 수록 GC도 빈번하게 발생하는 것

### 고려사항
1. 너무 작으면 GC가 자주 발생
2. 너무 크면 GC 한 번의 시간이 길어질 수 있음
3. 서버 전체 메모리를 고려
4. 컨테이너 환경에서는 메모리 제한 기준
   
   Docker나 Kubernetes에서는 물리 서버 RAM이 아니라 컨테이너 limit 기준으로 계산

5. heap 실행 중 변화가 적어야 함
   > Heap메모리가 부족하면, 최대 Heap메모리 사이즈가 될때까지 추가적으로 메모리 사이즈를 할당
   >

### 좋은 방법
1. Xms와 Xmx 동일하게 설정
   그 크기는 전체 서버 메모리의 50~70% 정도를 JVM Heap으로 설정하는 것이 일반적

2. 비율로 메모리 사이즈 설정

   서버의 스펙이 모든 상황에서 동일하다면, 굳이 비율(Percentage)로 메모리 사이즈를 설정해줄 필요는 없겠지만, 실제로는 매우 다양한 서버에서 동일한 서버 애플리케이션이 구동

   구동되는 서버마다 메모리 사이즈를 설정하는 것을 피하고, 어떤 서버에서나 동일한 수준의 메모리를 할당하고 싶다면, Heap메모리 사이즈를 비율로 설정

    1. -XX:InitialRAMPercentage
       ```bash
       java -XX:InitialRAMPercentage=50.0 -jar application.jar
       ```

       초기 Heap메모리 사이즈를 물리 메모리에 대한 비율로 설정하는 옵션
   2. -XX:MinRAMPercentage
      ```bash
      java -XX:MinRAMPercentage=50.0 -jar application.jar
      ```
      최대 메모리 사이즈를 지정하는 옵션
      
        서버가 256MB 미만 메모리 갖고 있을때만 동작
   3. -XX:MaxRAMPercentage
      ```bash
      java -XX:MaxRAMPercentage=50.0 -jar application.jar
      ```
      최대 Heap메모리 사이즈를 지정하는 옵션
      
        서버가 256MB 이상의 메모리를 갖고 있는 경우 동작

### JVM 메모리 옵션인 Xms와 Xmx의 의미와 설정 시 고려해야 할 사항은 무엇인가요?
> Xms는 JVM이 시작할 때 확보하는 초기 Heap 메모리 크기이고, Xmx는 JVM이 사용할 수 있는 최대 Heap 메모리 크기입니다. 
> 
> 설정할 때는 Heap 크기가 너무 작으면 GC가 자주 발생해 성능이 떨어지고, 너무 크면 GC 한 번에 걸리는 시간이 길어져 Stop-the-World 시간이 증가할 수 있다는 점을 고려해야 합니다. 
> 
> 또한 JVM은 Heap만 사용하는 것이 아니라 Metaspace, Thread Stack, Direct Memory, OS 메모리도 사용하기 때문에 서버 전체 메모리를 모두 Heap에 할당하면 안 됩니다.
> 
> 운영 환경에서는 Heap 크기가 실행 중 변동되는 오버헤드를 줄이고 GC 튜닝을 예측 가능하게 하기 위해 Xms와 Xmx를 동일하게 설정하는 경우가 많습니다. 
> 
> 컨테이너 환경에서는 물리 서버 메모리가 아니라 컨테이너 메모리 limit 기준으로 설정해야 하며, 여러 환경에서 같은 애플리케이션을 배포한다면 MaxRAMPercentage 같은 비율 기반 옵션도 고려할 수 있습니다.
