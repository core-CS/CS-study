## 자바에서 클래스는 단일 상속만 지원하지만, 인터페이스는 다중 구현이 가능한 이유?

> 자바에서 클래스 다중 상속을 금지하는 이유는 부모 클래스에 동일 이름의 구현 메서드가 있을 경우, 자식 클래스에서 어떤 것을 실행할 지 결정이 불가하기 때문입니다.
>
> 인터페이스는 추상적인 선언만 있어, 여러 인터페이스를 구현하더라도 결국 실제 로직은 구현 클래스에서 정의하기에 모호하지 않습니다.
>
> 동일 default method 로 정의되어 있더라도, 자바는 컴파일 시점에 명시적인 오버라이딩을 강제하기에 해결이 가능합니다.


### 🚨 클래스 다중 상속의 문제

바로 다이아몬드 문제이다.

자바가 클래스 단일 상속만 가능한 이유는 **모호성** 때문이다.

> 만약 `Class A`에 `doSomething()`메서드가 있고, 이를 상속받은 `Class B`와 `Class C`가 각각 `doSomething()`을 서로 다르게 오버라이딩 했다고 합시다.

이 때 `classD`가 B와 C를 동시에 상속 받게 된다면, `D` 기 `doSomething()` 을 호출했을 때 어떤 것을 실행해야 할까요?

자바에서는 컴파일러가 이를 판단할 수 없기에, 설계 단계에서 단일 상속만 허용합니다.

### 😒 인터페이스에서는 왜 다중 구현이 가능할까?

인터페이스는 기본적으로 메서드의 구현부가 없습니다.

만약 `Interface B` `Interface C` 에 동일한 `doSomething()` 이 선언되어 있더라도, 이를 구현하는 `class D` 는 어짜피 자신이 직접 메서드를 구현해야 합니다.

실행 시점에 누구의 것을 쓸 지 고민할 필요가 없고, 그냥 본인이 새로 정의한 로직이 실행될 것 입니다.

### 그렇다면 Default Method는?

우리가 인터페이스에 로직을 구현할 수 있도록 한 것이 default method 였습니다.

만약 두 인터페이스가 동일 시그니처의 default Method를 가지고 있고, 한 클래스가 이를 동시에 구현한다면 자바 컴파일러가 에러를 냅니다.

개발자는 반드시 구현 클래스에서 해당 메서드를 직접 오버라이딩하여 어느 인터페이스의 로직을 쓸지 직접 명시해야 합니다.

예: `InterfaceB.super.doSomething()`
