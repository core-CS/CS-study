<h2>Redis는 어떤 식으로 저장되는지 설명해 주세요.</h2>

>Redis는 인메모리 Key-Value 저장소다. 모든 데이터를 메모리(RAM)에 저장하기 때문에 디스크 기반 DB보다 훨씬 빠른 속도를 낼 수 있다.
>
> 저장 구조는 Key-Value 형태이지만, Value로 단순 문자열뿐 아니라 다양한 자료구조를 지원하는 것이 특징이다. 대표적으로 String, List, Set, Sorted Set, Hash가 있고, 그 외에 Bitmap, HyperLogLog, Stream, Geo 같은 특수 자료구조도 지원한다.
>
>메모리에만 저장하면 서버가 꺼졌을 때 데이터가 날아가기 때문에, Redis는 두 가지 영속성(Persistence) 방식을 제공한다. 
> 
>첫째는 RDB로, 특정 시점의 메모리 상태를 스냅샷으로 디스크에 저장하는 방식이다. 
> 
>둘째는 AOF(Append-Only File)로, 실행된 모든 쓰기 명령을 로그처럼 파일에 기록하는 방식이다. 실무에서는 둘을 혼용하는 경우가 많다.
>
>또한 Redis는 싱글 스레드 기반으로 동작하지만, Non-blocking I/O와 이벤트 루프를 활용해 높은 처리량을 보장한다. 주로 캐시, 세션 저장소, 실시간 랭킹, Pub/Sub 메시지 브로커 용도로 많이 쓰인다.

<h3>Redis 특징</h3>
1. 인메모리 저장  
    
2. 싱글 스레드
    멀트 스레드가 아닌데도 빠른 이유는 다음과 같다

   - 컨텍스트 스위칭 비용이 없음
   
   - 락(Lock) 경합이 없음
   
   - Non-blocking I/O(I/O 멀티플렉싱) 활용

   단, Redis 6.0부터는 네트워크 I/O 부분은 멀티 스레드를 지원한다고 한다.

3. 다양한 자료구조 지원
    
<h3>Redis의 영속성</h3>
1. RDB(Redis Database)
   
    특정 시점의 메모리 스냅샷을 바이너리 파일로 저장하는 방식
   
    장점: 파일 크기가 작고, 복구 속도가 빠르다. 백업용으로 적합
   
    단점: 스냅샷 사이의 데이터는 유실될 수 있다


2. AOF(Append-Only File)
   
    실행된 모든 쓰기 명령을 파일에 순서대로 기록하는 방식

    장점: 데이터 손실이 거의 없다. 더 안전 

    단점: 파일 크기가 크고, 복구 속도가 RDB보다 느리다
